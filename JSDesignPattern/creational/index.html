Creational Design Pattern

Constructor Pattern

Use to create new objects with their own object scope

New Keyword

Example


				function ObjectName(params) {
					this.params = params;
					this.someFunction = function() {
						return this.params;
					}
					// new keyword would add "return this;"
				}

Drawback: this.someFunction is created everytime a Object is created

Solution: Use Prototype

Prototype/Class

Prototype Example


				ObjectName.prototype.methodName = function() {};
				// This prevents more than one creation of the same function

Class Example


				'use strict'
				// has to be in strict mode

				class MyObject {
					constructor (params) {
						this.params = params;
					};

					myMethod () {
						return "something";
					};
				}

Module Pattern

Simple way to encapsulate methods --> create a toolbox

Example


				var Module = {
					method: function () {},
					nextMethod: function () {}
				}

Module Pattern: We need only one service/module for my application
